  • Principia is a modeling and simulation tool for educators and students of kinematics. We believe that having visualizations that are flexible and packed with information can help students better understand material presented in a Physics I class.   • Quick Start Guide

    Use the components in the toolbox tab to model a physics problem. After dragging it on to the canvas, you can select an object and the properties tab will open. From there, you can modify properties such as velocity, acceleration, mass, spring constants, and more. The simulator automatically generates 1000 frames of motion by default.


    You can enable more advanced features by either finding the toggle in the properties tab, right-clicking an object and using the context menu that appears, or by using shortcut keys.


    Current shortcuts include:

    • f: Pause the playback and draw a free-body diagram of the currently selected body
    • a: Add acceleration to currently selected body (towards cursor)
    • v: Add velocity to currently selected body (towards cursor)
    • del: Delete currently selected body


    If you want to solve for unknown values, you will need to add additional keyframes using the keyframe tab. A "keyframe" is a snapshot in time representing state changes in a problem. It may represent the instant that something beings accelerating, or the instant it comes to rest. Once multiple keyframes exist, the properties window will have a new icon of a shrugging man that you can click to indicate you don't know the value of a certain property. When you click play (and assuming there are enough values that are known), these unknown values will be filled in with an analytic solution. You can use the solution tab to see the details.


    Use the globals tab to edit properties that affect the entire simulation, such as where the origin is (object coordinates are displayed relative to this) or which unit to use.


    Use the overview tab for a quick list of all of the objects in your scene. You can click an object in the overview to select that object and center the camera on it, or you can click the garbage bin to delete that object.


    Why don't the numbers match my homework?!

    Principia is built in two parts: the sandbox where you model your problem and a solver that attempts to fill in unknown values.


    If you only have one keyframe, your problem does not need the solver. A single keyframe describes an initial state where all values are known. The next 999 frames are generated by calculating the forces acting on each body and modifying the acceleration, velocity, and position accordingly. However, since Principia works using discrete frames, a fixed amount of time passes per frame. Error can accumulate as a result. For example, the force of a spring pulling on a body is constantly changing as it oscillates. Principia computes the value of that force at two snapshots in time, not accounting for the minute changes in force that occur between those frames.


    Therefore, you can use the sandbox mode to examine general principles and gain an overall feel for how bodies might interact in the real world, but it does not yield exact values.


    Thanks to the existence of the solver, however, Principia is still potentially useful for problems where you do need to know an exact value. The solver is not concerned with the error that accumulates on the in-between frames because it does a first pass using the fundamental kinematics equations, filling in the unknowns that you specify in each keyframe. Non-keyframes are still approximations, but as soon as the simulator reaches a keyframe (indicated by a blue highlight in the keyframes tab), the delta-time and object properties are all adjusted to match the exact values that are analytically solved for before the playback begins.


    If you are using Principia to better understand your homework, you should open the solution tab to see how that analytic solution was computed. You might regret it if you simply try to copy answers without understanding the equations. Your instructor certainly doesn't plan to let you have access to Principia when you are taking your final!


    The solver does not currently account for more advanced physics equations, nor does it respond to the fact that bodies may collide on in-between frames. Keep an eye out for future updates addressing these issues, or feel free to help contribute yourself!
